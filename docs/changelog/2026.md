## 2026-02-03 {small}`nf 0.5.0`

This release revamps the configuration (`nextflow.config`) for more control over which artifacts are tracked and which metadata is attached (Kind, Project, ULabels). Some old config options and functions are deprecated and will be removed in 0.6.0.

Example nextflow config:

```groovy
lamin {
  instance = "laminlabs/lamindata"
  api_key = secrets.LAMIN_API_KEY
  output_artifacts {
    rules {
      output {
        pattern = ".*"
        order = 1
        type = "include"
        project_uids = ["AbCdEfGhIjKl"]
        ulabel_uids = ["AbCd0123"]
      }
      // don't publish state.yaml files
      yaml {
        pattern = ".*\\.state\\.yaml"
        type = "exclude"
        order = 10
      }
    }
  }
}
```

New functionality:

- âœ¨ **Advanced Artifact Tracking**: Control which files are tracked using pattern-based rules, and which metadata is attached to the artifacts ([#121](https://github.com/laminlabs/nf-lamin/pull/121)).

- âœ¨ **Metadata Linking**: Link projects and ulabels to transforms, runs, and artifacts. New `run` and `transform` config sections for granular control ([#121](https://github.com/laminlabs/nf-lamin/pull/121)).

- âœ¨ **Restructured API Configuration**: New `api` section for connection settings replaces root-level fields ([#121](https://github.com/laminlabs/nf-lamin/pull/121)).
